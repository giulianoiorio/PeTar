Detailed Description

Evolve functor specialised for the analysis of stages of binary lifetime with a compact object plus a luminous variable (BLC). The evolution starts record the binary states (following the dtout option) when we enter in a phase with an unbroken binary containing a compact object plus a luminous star (Phase<7 or WD). The evolution is broken when a broken trigger is true or after we conclude a BLC phase since there is no way that a new BLC phase can occur. @Note The dtout end option means that only the start and the end of the BLC phase are printed; @Note This evolution can be used only with binaries, using it with stars raise a not implemented error. Evolve functor specialised for the analysis of stages of binary lifetime with a compact object plus a luminous variable (BLC). @record_state_policy The states start to be recorded when the binary enters the first time in a BLC phase (bound compact + Luminous star) The last record is got the first time the binary exits the BLC phase (e.g. the Luminous star becomes a compact object or a merger occurs). Inside this window the state are recorded following the dtout option. When dtout=end only the first (start of the BLC) and last (end of the BLC) states are recorded. @break_evolve_policy TThe evolution is broken when a broken trigger is true (depending on the input tf) or after we conclude a BLC phase since there is no way that a new BLC phase can occur. @binstar_evolve Yes @star_evolve raise Not implemented error @catch sevnerr
